# Project 11 — IAM Governance & Least Privilege Framework

## Project Overview
This project is focused on designing and building a reusable AWS IAM governance framework that enforces least privilege access. The goal is to improve security and control over AWS identities, roles, and permissions.

This project is currently in development.

---

## Planned Features

- Role-based access control (RBAC)
- Permission boundaries for IAM users and roles
- AWS IAM Access Analyzer integration
- Cross-account role assumption
- Temporary credentials using AWS STS

---

## Objectives

- Understand and implement least privilege IAM design
- Reduce overly permissive IAM policies
- Apply secure cross-account access patterns
- Use temporary credentials instead of long-term access keys

---

## Technologies (Planned)

- AWS IAM
- AWS STS (Security Token Service)
- AWS IAM Access Analyzer
- AWS Organizations (optional multi-account setup)
- AWS CLI
- Infrastructure as Code (Terraform or CloudFormation - optional)
